one grid for all your simulators

simgrid

Run multiple Expo / React Native projects on multiple simulators, emulators and devices — in parallel, without the alt-tab dance.

$ npm i -g simgrid-cli View on npm → GitHub

The problem

Working on several Expo apps at once is painful: launch one and it grabs whichever simulator it feels like — often the wrong one, or one already claimed by another project. You alt-tab between windows, restart Metro, resolve port conflicts by hand, and lose track of which app runs where. simgrid solves this at the device layer.

What it does

🎛️

Interactive picker

Pick one or many simulators, emulators or physical devices. Your last choice is pre-checked.

🚀

Parallel launch

Run the same project on several devices at once — one Metro per project, reused across all of them.

🔍

Dev-build aware

Detects which devices already have your dev build for instant launch; builds the rest, remembering how.

🔌

No port conflicts

Allocates a free Metro port per project and deep-links the Expo dev client automatically.

🧭

Knows who runs what

A shared, self-healing registry tracks every session and survives crashes & kill -9.

📋

Logs · doctor · profiles

Stream device logs, check your toolchain, and replay saved device sets in one command.

Commands

CommandWhat it does
simgridInteractive picker, then launch (the default).
simgrid --profile <name>Launch a saved device set in one shot — saved automatically the first time you use a new name.
simgrid initWire "start": "simgrid" into this project's package.json.
simgrid statusShow which project runs on which device.
simgrid logs [device]Stream a running device's system logs; picker when several are running.
simgrid profilesList the saved device profiles for this project.
simgrid doctorCheck that xcrun / adb / emulator are installed, with install hints.
simgrid stopStop this project's sessions and deregister them.

How it works

  1. Read project identity from app.json / app.config.ts (name, scheme, bundle ID).
  2. Load the shared registry (~/.simgrid/state.json) and reconcile it against live PIDs.
  3. Discover every simulator, emulator and physical device; annotate build installed? and busy?
  4. Show the picker — pre-checked with your last choice for this project.
  5. Boot, start one Metro per project, and deep-link the dev client (or build first if needed).
  6. Register the session; deregister cleanly on Ctrl-C or Metro exit.

Get started

# install (published as "simgrid-cli"; the command is "simgrid")
npm i -g simgrid-cli

# in any Expo project — wire it into npm start (optional)
simgrid init

# or just run it directly
simgrid

Requires Node ≥ 18. iOS simulators need macOS + Xcode; Android needs adb & emulator on your PATH. Your project should use an Expo development build.
