One grid for all your simulators.

$ npm i -g simgrid-cli

Name

simgrid — run multiple Expo projects on multiple devices, in parallel.

Synopsis

simgrid [command] [--profile name]

Description

Juggling several Expo apps means each one grabs whichever simulator it wants — often the wrong one, or one already taken. You alt-tab, restart Metro, fix port clashes by hand.

simgrid routes each project to the right device, runs one Metro per project, and remembers who runs what. No daemon, no config. It works across iOS simulators, Android emulators and physical devices, detects which already have your dev build, and frees you from the alt-tab dance.

Commands

simgrid
Interactive picker, then launch — the default.
simgrid --profile <name>
Launch a saved device set; saved automatically the first time you use a name.
simgrid init
Wire "start": "simgrid" into the project's package.json.
simgrid status
Show which project runs on which device (global).
simgrid logs [device]
Stream a running device's system logs; picker when several run.
simgrid profiles
List saved device profiles for this project.
simgrid doctor
Check xcrun / adb / emulator on your PATH, with install hints.
simgrid stop
Stop this project's sessions and deregister them.

Use cases

Several client apps at once
agencyBoot 3 projects on 3 devices in three tabs. Each gets its own Metro and port — one glance at simgrid status.
One app, many devices
qaLaunch the same project on an iPhone, an iPad and a Pixel in parallel to check a layout across sizes and OS versions.
iOS and Android side by side
cross-platformBuild a feature once and watch it live on both platforms at the same time.
Replay your exact setup
demoSave a device set once with simgrid --profile demo; next time it relaunches instantly, no picker.

Examples

Pick devices & launch

$ simgrid

  simgrid  Yolgo
  iOS Simulators
  › [x] iPhone 15            ✅ build installed
    [x] iPhone SE (3rd gen)  ⚙️ will build
    [ ] iPhone 15 Pro        ✅ build installed · 🔴 busy: shoootin :8082

▶ Metro for Yolgo on port 8081
  iPhone 15: dev client → localhost:8081 ✓
✔ Yolgo — Ctrl+C to stop

Who runs where

$ simgrid status
● Yolgo → iPhone 15 (Metro :8081, pid 41201)
● Yolgo → iPhone SE (3rd gen) (Metro :8081, pid 41201)
● shoootin → Pixel 7 (Metro :8082, pid 41588)
● mdev2 → iPhone 15 — simgrid (Metro :8083, pid 41922)

Install

# published as "simgrid-cli" — the command is "simgrid"
npm i -g simgrid-cli

# wire it into npm start (optional), or just run it
simgrid init
simgrid

Requires Node ≥ 18. iOS sims need macOS + Xcode; Android needs adb & emulator on PATH. Use an Expo development build.
