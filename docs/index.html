sim·grid

One grid for all your simulators. Run multiple Expo projects on multiple devices — in parallel, without the alt-tab dance.

$ npm i -g simgrid-cli npm → GitHub

00 / The idea

Juggling several Expo apps means each one grabs whichever simulator it wants — often the wrong one, or one already taken. You alt-tab, restart Metro, fix port clashes by hand. simgrid routes each project to the right device, runs one Metro per project, and remembers who runs what. No daemon. No config. Works across iOS simulators, Android emulators and physical devices.

01 / Use cases

UC-01 · AGENCY

Several client apps at once

Boot 3 projects on 3 devices in three terminal tabs. Each gets its own Metro and port — zero collisions, one glance at simgrid status.

UC-02 · QA

One app, many devices

Launch the same project on an iPhone, an iPad and a Pixel in parallel to check a layout across screen sizes and OS versions in one shot.

UC-03 · CROSS-PLATFORM

iOS + Android side by side

Build a feature once and watch it live on both platforms simultaneously — no more launch-kill-relaunch between iOS and Android.

UC-04 · DEMO

Replay your exact setup

Save your demo device set once: simgrid --profile demo. Next time it relaunches the same devices instantly, no picker.

03 / Commands

CommandFunction
simgridInteractive picker, then launch — the default.
simgrid --profile <name>Launch a saved device set; saved automatically the first time you use a name.
simgrid initWire "start": "simgrid" into the project's package.json.
simgrid statusShow which project runs on which device (global).
simgrid logs [device]Stream a running device's system logs; picker when several run.
simgrid profilesList saved device profiles for this project.
simgrid doctorCheck xcrun / adb / emulator on your PATH, with install hints.
simgrid stopStop this project's sessions and deregister them.

04 / How it works

  1. Read project identity from app.json / app.config.ts (name, scheme, bundle ID).
  2. Load the shared registry ~/.simgrid/state.json and reconcile it against live PIDs.
  3. Discover every simulator, emulator and device; mark build installed? and busy?
  4. Show the picker, pre-checked with your last choice for this project.
  5. Boot, start one Metro per project, deep-link the dev client (or build first if needed).
  6. Register the session; deregister cleanly on Ctrl-C or Metro exit.

05 / Get started

install
# published as "simgrid-cli" — the command is "simgrid"
npm i -g simgrid-cli

# wire it into npm start (optional)
simgrid init

# or just run it, in any Expo project
simgrid

Requires Node ≥ 18. iOS sims need macOS + Xcode; Android needs adb & emulator on PATH. Use an Expo development build.
